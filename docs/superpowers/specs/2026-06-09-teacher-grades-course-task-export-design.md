# 教师端学生成绩课程化与作业分数导出设计

## 1. 目标

在教师端“学生成绩”页面补齐课程维度和作业分数导出能力：

- 作业列表按课程查看，并在作业名称后展示课程名称。
- 保留现有作业二级详情抽屉及其导出能力。
- 学生成绩 Excel 导出按课程拆分工作表，每个工作表展示该课程下每名学生每次作业的分数。

## 2. 当前现状

`frontend/src/views/teacher/TeacherStudents.vue` 当前包含：

- 顶部任务概览卡片。
- 作业列表，点击后打开 `TeacherTaskDetail.vue`。
- 班级筛选、学生搜索、学生成绩表格。
- 调用 `/api/teacher/students/export` 导出学生成绩 Excel。

后端现状：

- 作业概览来自 `backend/app/services/task_service.py` 的 `task_overview`，接口为 `/api/announcements/task-overview`。
- 作业详情分数来自 `completion_report`，接口为 `/api/announcements/{announcement_id}/completion-report`。
- 学生成绩列表和 Excel 导出来自 `backend/app/services/teacher_service.py` 与 `backend/app/api/v1/routes/teacher_routes.py`。
- 当前 Excel 只有学生基础信息、已完成数、未完成数和完成率，缺少每次作业分数列。

## 3. 范围

### 3.1 本次纳入

- 在学生成绩页增加课程筛选控件，用于筛选作业列表和学生成绩数据。
- 作业卡片标题显示为“作业名称（课程名称）”，避免同名作业无法区分。
- `task_overview` 返回每个作业的 `course_id`、`course_name`，并支持按课程筛选。
- `/api/teacher/students` 支持 `course_id` 筛选，筛选该课程下班级学生及该课程任务完成统计。
- `/api/teacher/students/export` 支持课程化导出：
  - 未选择课程时：每个课程一个 Sheet，另保留总览 Sheet。
  - 选择课程时：导出该课程一个 Sheet。
  - 每个课程 Sheet 的列为学生基础信息 + 该课程下每次作业分数。
- 后端新增针对导出内容和课程过滤的回归测试。
- 前端 API 类型同步新增课程字段和筛选参数。

### 3.2 本次不纳入

- 不改作业详情抽屉的 CSV 导出格式。
- 不新增独立的作业成绩页面。
- 不调整学生端答题流程、计分规则、任务完成规则。
- 不引入新的前端状态管理或 Excel 生成库。

## 4. 方案取舍

### 方案 A：按课程拆 Sheet（推荐）

Excel 每个课程一个工作表，行是学生，列是该课程下每次作业分数。页面也按课程筛选查看作业。

优点：

- 符合教师按课程管理成绩的习惯。
- 避免所有课程作业横向堆在同一个 Sheet 导致过宽。
- 和项目“课程 -> 班级 -> 学生/作业”的长期结构一致。

缺点：

- 后端导出需要一次性聚合课程、班级、学生、任务和答题分数。

### 方案 B：单 Sheet 横向展开全部作业

全部学生放在一个 Sheet 中，所有课程的作业都作为列。

优点：文件结构简单。

缺点：课程和作业多时可读性差，同名作业难区分，后续筛选成本高。

### 方案 C：沿用当前总览导出，只补课程名称

只在当前 Excel 中增加课程信息，不导出每次作业分数。

优点：改动最小。

缺点：不能满足“每次作业和每次作业的分数”的核心需求。

最终采用方案 A。

## 5. 页面设计

### 5.1 筛选区

学生成绩页筛选区增加“全部课程”下拉框，数据来源复用课程列表接口。

筛选行为：

- 选择课程后，作业列表只展示该课程作业。
- 选择课程后，班级下拉框只展示该课程下班级。
- 选择课程后，学生列表只展示该课程下班级学生。
- 清空课程后，恢复全部课程视图。

### 5.2 作业列表

作业卡片标题展示为：

```text
作业标题（课程名称）
```

卡片下方仍展示目标班级、完成数、总人数和任务状态。

### 5.3 Excel 导出按钮

按钮文案保持简洁：

- 未筛选：`导出全部学生`
- 筛选课程：`导出当前课程`
- 筛选班级：`导出当前班级`

如果同时筛选课程和班级，以班级为更窄范围，但导出的 Sheet 仍包含对应课程下该班级学生的每次作业分数。

## 6. 后端数据设计

### 6.1 作业概览

`task_overview(db, teacher_id, course_id=None)` 增加可选课程参数，并在每个任务中返回：

```json
{
  "id": 1,
  "title": "课程练习",
  "course_id": 1,
  "course_name": "测试课程",
  "class_names": ["2025级1班"],
  "total_students": 30,
  "completed_count": 20,
  "is_expired": false,
  "created_at": "2026-06-09T10:00:00"
}
```

### 6.2 学生列表

`list_students(db, teacher_id, class_id=None, page=None, page_size=None, keyword=None, course_id=None)` 增加可选课程参数。

过滤规则：

- `course_id` 必须属于当前教师创建的课程，否则返回空列表。
- 选择课程时，只统计该课程下的班级、学生和作业。
- 同时传入 `class_id` 时，班级必须属于该课程且属于当前教师。

### 6.3 Excel 作业分数

新增服务函数用于构建导出数据，避免在路由中堆复杂查询：

```python
build_student_task_score_export(db, teacher_id, course_id=None, class_id=None)
```

返回结构按课程分组：

```python
[
    {
        "course_id": 1,
        "course_name": "测试课程",
        "tasks": [
            {"id": 10, "title": "第一次作业"},
            {"id": 11, "title": "第二次作业"},
        ],
        "students": [
            {
                "serial_no": 1,
                "id": "2025001",
                "name": "张三",
                "major": "人工智能",
                "class_name": "2025级1班",
                "scores": {10: 100, 11: 80},
            }
        ],
    }
]
```

计分规则沿用 `completion_report`：

- 分数按任务上下文统计。
- 每题取该任务下最新一次答题结果。
- 分数为百分制整数。
- 未答或未完成但有部分答题时，按已有最新答题结果计算；完全未答显示空值。

本次设计选择“完全未答显示空值”，避免把“没有提交”误读为“得 0 分”。如学生有答题记录但未满分，则按实际答题正确率显示分数。

## 7. Excel 格式

未选择课程时：

- Sheet 1：`总览`
- 后续 Sheet：每个课程一个 Sheet，Sheet 名使用课程名，超过 Excel 限制时截断。

课程 Sheet 表头：

```text
序号 | 学号 | 姓名 | 专业 | 班级 | 已完成 | 未完成 | 完成率(%) | 第一次作业 | 第二次作业 | 后续作业列
```

每个作业列的表头使用：

```text
作业标题
```

如果同一课程下存在同名作业，表头追加任务 ID：

```text
作业标题 #12
```

## 8. 错误处理

- 教师传入不属于自己的 `course_id` 或 `class_id` 时，列表返回空数据，导出返回只含表头或空总览，不暴露其他教师数据。
- 没有课程、班级或学生时，前端保持现有空状态文案。
- 导出失败时，前端仍提示“导出失败，请稍后重试”。

## 9. 验收标准

- 学生成绩页可以按课程筛选作业列表。
- 作业卡片可以看到课程名称。
- 按课程筛选后，班级选项只剩该课程下班级。
- 学生列表统计只包含当前课程范围。
- 导出的 Excel 是 `.xlsx`。
- 未选择课程时，Excel 至少包含 `总览` 和每个课程 Sheet。
- 每个课程 Sheet 包含该课程下每次作业的分数列。
- 二级作业详情抽屉仍可打开，原导出能力不被移除。
- 后端测试覆盖课程过滤、Excel Sheet 和作业分数列。
- 前端构建通过。
